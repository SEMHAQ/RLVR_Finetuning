# GRPO 数学推理增强：规则反馈与多源反馈的效果分析

## Research Plan (v2 — 基于实验结果重构)

---

## 一、研究背景与动机

### 1.1 问题场景：数学推理的强化学习微调

大语言模型（LLM）的数学推理能力是智能教育系统的核心。GRPO（Group Relative Policy Optimization）作为 DeepSeek-R1 提出的强化学习算法，通过组内相对优势估计实现了无需 critic model 的高效策略优化。然而，GRPO 在数学推理任务上的实际效果、最优训练配置、以及多源反馈信号的融合效果仍缺乏系统性研究。

### 1.2 研究问题

1. **GRPO 在数学推理上的有效性**：在 Qwen2.5-Math-1.5B 上，GRPO 能带来多大的性能提升？
2. **训练配置的影响**：学习率、LoRA vs 全量微调、数据规模等因素如何影响 GRPO 效果？
3. **多源反馈融合的效果**：在规则反馈（答案正确性）基础上，加入 AI 反馈（推理质量）和标量反馈（表达规范性）是否能进一步提升性能？

### 1.3 核心发现（实验驱动）

通过大量实验，我们发现：

- **GRPO 效果显著**：纯规则反馈的 GRPO 在 GSM8K 上从 44.28% 提升到 79.38%（+35.1%），在 MATH-500 上达到 53.8%
- **多源反馈反而降低性能**：加入 AI 反馈和标量反馈后，GSM8K 降至 69.75%~72.55%，MATH-500 降至 48.0%
- **原因分析**：在答案可验证的任务中，规则反馈已经是完美信号，辅助反馈引入的是噪声而非有用信息

## 三、方法设计

### 3.1 GRPO 基础框架

**标准 GRPO 算法**（DeepSeek-R1）：

对每个 prompt $x$，采样一组 response $\{y_1, ..., y_G\}$，计算组内相对优势：

$$\hat{A}_i = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})}$$

策略梯度更新使用 PPO-clip 目标：

$$\mathcal{L}_{GRPO} = -\mathbb{E}\left[\min\left(\frac{\pi_\theta}{\pi_{old}} \hat{A}, \text{clip}\left(\frac{\pi_\theta}{\pi_{old}}, 1-\epsilon, 1+\epsilon\right) \hat{A}\right)\right] + \beta \cdot D_{KL}(\pi_\theta \| \pi_{ref})$$

**规则反馈（Rule-based Reward）**：

对于数学推理任务，答案可由规则验证：

$$r_{rule}(x, y) = \begin{cases} 1.0 & \text{if extract\_answer}(y) = \text{ground\_truth}(x) \\ 0.0 & \text{otherwise} \end{cases}$$

### 3.2 MSF-GRPO：多源反馈融合

在 GRPO 框架中融合三种反馈信号：

```
completions → [规则反馈] → r_rule (0/1)
           → [AI反馈]   → r_ai (0-1, 规则模拟)
           → [标量反馈] → r_scalar (0-1)
                              ↓
                    Adaptive Fusion → r_fused
```

**AI 反馈**（规则模拟 LLM-as-judge）：
- 基于步骤数量、计算过程、推理连贯性评分（0-1）
- 无需额外模型，零显存开销

**标量反馈**（格式 + 步骤质量）：
- 格式分 (0-0.3)：是否有 `####`、是否分步骤
- 步骤完整度 (0-0.4)：推理步骤数量、是否有计算过程
- 表达清晰度 (0-0.3)：结构化程度

**自适应融合**（方差感知加权）：

$$w_i = \frac{\rho_i / \sigma_i^2}{\sum_j \rho_j / \sigma_j^2}$$

- $\rho_{rule}=1.0 > \rho_{ai}=0.8 > \rho_{scalar}=0.5$（可靠性先验）
- $\sigma_i^2$ 为当前 batch 的 reward 方差

**最终 reward**（gated aux）：

$$r_{fused} = r_{rule} + \alpha \cdot r_{aux} \quad \text{if } r_{rule} > 0 \text{ else } 0.0$$

仅在答案正确时给予辅助信号 bonus，避免错误答案被辅助信号"美化"。

## 四、实验设计与结果

### 4.1 实验设置

| 项目 | 配置 |
|------|------|
| 基础模型 | Qwen2.5-Math-1.5B |
| 训练框架 | TRL GRPOTrainer |
| 训练数据 | GSM8K train (7.5K samples) |
| 评测数据 | GSM8K test (1.3K) / MATH-500 (500) |
| 硬件 | RTX 3090 24GB |
| 微调方式 | LoRA (r=16, alpha=32) |
| 训练步数 | 934 steps (1 epoch) |

### 4.2 主实验结果

| 实验 | 方法 | 学习率 | GSM8K | MATH-500 |
|------|------|--------|-------|----------|
| baseline | 无训练 | — | 44.28% | — |
| grpo_v1 | 全量微调 | 5e-7 | 44.20% | — |
| grpo_v2 | LoRA (1K 样本) | 2e-5 | 44.66% | — |
| **grpo_v3** | **LoRA (全量)** | **2e-5** | **79.38%** | **53.8%** |
| grpo_msf | MSF-GRPO (α=0.2) | 5e-6 | 72.55% | — |
| grpo_msf | MSF-GRPO (α=0.05) | 5e-6 | 69.75% | 48.0% |

### 4.3 关键发现

**发现 1：GRPO 在数学推理上效果显著**

- 从 44.28% 到 79.38%，绝对提升 35.1 个百分点
- 1.5B 模型经过 GRPO 训练后，接近 Qwen2.5-Math-7B 的 few-shot 水平（~84%）
- 证明 GRPO + 规则反馈是数学推理的有效微调方法

**发现 2：LoRA 优于全量微调（在小学习率下）**

- 全量微调 lr=5e-7 效果无提升（44.20%）
- LoRA lr=2e-5 效果显著（79.38%）
- 可能原因：全量微调学习率过低，LoRA 的参数效率允许使用更高学习率

**发现 3：多源反馈融合反而降低性能**

| 方法 | GSM8K | MATH-500 | 分析 |
|------|-------|----------|------|
| 纯规则 GRPO | 79.38% | 53.8% | 基线 |
| MSF-GRPO (α=0.2) | 72.55% (-6.83%) | — | 辅助信号噪声过大 |
| MSF-GRPO (α=0.05) | 69.75% (-9.63%) | 48.0% (-5.8%) | 即使小权重仍有害 |

**原因分析**：
- 在答案可验证的任务中，规则反馈（0/1）是完美信号
- AI 反馈和标量反馈基于启发式规则，与正确答案的相关性弱
- 即使使用 gated aux（仅在正确时加 bonus），辅助信号仍在组内相对优势计算中引入噪声
- GRPO 的组内归一化放大了辅助信号的方差影响

**发现 4：训练数据规模的影响**

- 1K 样本 → 44.66%（几乎无提升）
- 7.5K 样本 → 79.38%（显著提升）
- 说明 GRPO 需要足够的数据多样性来建立有效的组内对比

### 4.4 消融实验（计划）

| 消融项 | 目的 | 状态 |
|--------|------|------|
| 不同学习率 (1e-6, 5e-6, 1e-5, 2e-5) | 找到最优 lr | 计划中 |
| 不同数据规模 (1K, 3K, 5K, 7.5K) | 数据效率分析 | 计划中 |
| 不同 num_generations (2, 4, 8) | 采样数影响 | 计划中 |
| 纯 AI 反馈 vs 纯标量反馈 | 单独辅助信号效果 | 计划中 |
